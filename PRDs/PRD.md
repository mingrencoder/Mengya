# “萌芽”个人生活记录平台 - 产品需求文档 (PRD)

## 1. 产品概述

**产品名称：** 萌芽
**产品定位：** 一个极具设计感的个人生活与灵感记录空间。
**核心价值：** 为站长提供一个纯粹、沉浸式的个人展示与记录平台，打造极具个人辨识度的专属网页，同时向访客提供流畅、美观的视觉与交互体验。

## 2. 目标用户

* **管理员（站长本人）：** 需要一个便捷的后台来随时自定义主页的展示内容、发布生活游记和管理常用收藏。
* **访客（亲友、网友）：** 浏览站长展示的个性化主页、生活动态和推荐链接。

## 3. 核心功能需求

### 3.1 前台展示模块 (访客端)

1. **自定义个性化首页 (Home)：**
   * **核心展示区：** 突破固定的简介排版，支持站长高度自定义的图文展示模块。不仅可以展示个人头像、欢迎语，还可以灵活配置额外的自定义区块（例如：近期感悟、个人生活理念、置顶的精选照片等）。
   * **动态呈现：** 首页内容需具备平滑的载入动画，提升第一印象的质感。

2. **旅行记录 (Travel Log)：**
   * 以视觉优先的卡片流或时间轴形式展示旅行足迹。
   * 每条记录包含：地点名称、时间、一张高质量封面图。
   * 支持图片点击后的大图沉浸式预览（Lightbox 效果）。

3. **常用收藏 (Bookmarks)：**
   * 分类展示站长精选的工具网站、设计灵感等链接。
   * 采用网格卡片布局，卡片需清晰展示：网站名称、简短描述、外链跳转图标。

### 3.2 后台管理模块 (管理员端)

1. **安全认证：**
   * 提供受保护的唯一管理员登录入口，确保管理权限的绝对安全。

2. **主页自定义管理 (核心功能)：**
   * 提供直观的首页内容编辑能力。站长可以在后台自由修改首页的各项文字内容、替换展示图片，以及调整自定义区块的显示内容，实现前台首页的个性化常换常新。

3. **日常内容管理：**
   * **旅行管理：** 添加新的足迹（上传照片、填写地点和时间），支持对现有足迹的编辑和下架。
   * **收藏管理：** 维护书签库，支持新增分类，以及在各分类下添加、修改、删除链接记录。

4. **全局设置：**
   * 支持修改网站的基础公共信息，如站点副标题、页脚宣告等。

## 4. 界面与用户体验要求 (UI/UX)

1. **核心设计风格：**
   * 采用极简科技风（Linear 风格）结合苹果生态常见的毛玻璃质感（Glassmorphism）。
   * 必须采用**纯粹的暗黑模式 (Dark Mode)** 作为全局默认主题。

2. **视觉细节规范：**
   * **背景与光影：** 极深的纯色背景，局部搭配微弱的渐变环境光晕（Glow），赋予页面呼吸感。
   * **材质与层次：** 大量使用带有背景模糊效果的半透明面板和卡片，配合极细的半透明边框，拉开视觉层次。
   * **排版：** 极简的无衬线字体，高对比度的文字排版，确保在暗色背景下的极佳可读性。

3. **多端自适应 (响应式设计)：**
   * 网页必须完美适配各类终端设备。
   * **电脑端：** 采用经典的侧边栏导航 + 右侧宽大内容区布局。
   * **平板/手机端：** 侧边栏自动隐藏，转换为底部标签导航或顶部折叠菜单；内容网格自动调整为适合单手滑动的单列布局。
