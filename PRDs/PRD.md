# “萌芽”个人生活记录平台 - 产品需求文档

## 1. 产品概述

**项目名称：** 萌芽
**项目目标：** 开发一个支持浏览器/服务器架构的轻量级个人生活记录网页平台。系统部署于腾讯云服务器，采用无传统数据库的架构，通过本地文件加密存储确保数据安全与服务器资源的极低占用。前端需具备极高的设计感，并完美适配电脑、平板和手机端。
**核心价值：** 极简部署、极低维护成本、数据完全私有、极致的视觉体验。

## 2. 目标用户

* **管理员（站长本人）：** 需要一个安全的后台来随时随地发布、修改、删除个人内容。
* **访客（亲友、网友）：** 浏览站长的生活动态、旅行记录和分享的灵感书签，体验流畅、美观的交互。

## 3. 系统架构要求

### 3.1 前端架构

* **技术栈选型推荐：** React + Tailwind CSS（或 Vue3 + Tailwind CSS）。
* **设计风格：** **极简科技风格结合苹果风格的毛玻璃质感**。
  * **主题：** 纯粹的暗黑模式为主，使用极深的黑色背景。
  * **质感：** 大量使用带有背景模糊效果的半透明面板和卡片。
  * **点缀：** 背景增加动态的微弱渐变光晕，文字排版具备高对比度。
* **自适应要求：** **必须**采用响应式设计，完美适配电脑端（侧边栏布局）、平板端（适中网格）和手机端（底部导航或顶部菜单布局）。
* **单页面应用：** 页面切换需平滑过渡，提供无刷新体验。

### 3.2 后端架构

* **技术栈：** Node.js（推荐使用 Express 或 Koa 框架）。
* **数据存储策略（核心）：**
  * **无数据库：** 绝对不使用 MySQL 等任何关系型数据库。
  * **本地文件存储：** 数据以纯文本格式存储在服务器本地文件系统中。
* **加密机制：**
  * 后端在将数据写入本地文件前，必须使用对称加密算法进行加密。
  * 前端请求数据时，后端读取文件、解密后通过接口返回。
* **媒体文件处理：** 图片等文件直接上传至服务器的静态资源目录，提供静态文件访问服务。

## 4. 核心功能需求

### 4.1 前台展示模块 (访客视角)

1. **首页介绍：**
   * 展示个人头像、欢迎语、核心简介。
   * 阐述“萌芽”网站的技术架构亮点（如“轻量化”、“无数据库”）。

2. **旅行记录：**
   * 以卡片流或时间轴的形式展示旅行足迹。
   * 每条记录包含：地点名称、时间、一张封面图。
   * 图片需支持点击放大预览。

3. **常用收藏：**
   * 分类展示常用的工具网站、灵感链接。
   * 网格布局，卡片需包含：网站名称、简短描述、外链跳转图标。

### 4.2 后台管理模块 (管理员视角)

1. **安全登录：**
   * 唯一管理员账号密码登录。
   * 登录成功后获取身份令牌，后续请求需携带。
   * 必须包含防止暴力破解的机制。

2. **控制台：**
   * 展示系统运行状态（如后端服务状态、加密开启状态、数据文件大小等）。

3. **内容管理：**
   * **首页管理：** 修改简介文字、欢迎语。
   * **旅行管理：** 添加新足迹（上传图片、输入地点、时间）、编辑/删除现有足迹。
   * **收藏管理：** 增删改查书签链接及分类。
   * *注：所有管理操作最终都将触发后端文件的重写并自动加密。*

## 5. 非功能性需求

* **安全性：** 后端接口需做权限校验，防止未授权的写操作。存储在服务器的数据文件必须是加密状态，即使服务器文件被非法获取也无法读取明文。
* **性能：** 前端静态资源需压缩；接口响应时间需在极短时间内完成（考虑到本地文件读写极快）。
* **部署：** 需提供清晰的部署脚本，方便在腾讯云上快速上线。
